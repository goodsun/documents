# CLAUDE.md — Documents Vault 書き込みルール

このディレクトリは開発案件のドキュメントを蓄積する Obsidian Vault。Claude Code がここで作業するときは以下に従うこと。

## どこに何を書くか

| 内容 | 書き込み先 |
|------|-----------|
| 調査結果・デバッグ過程・試行錯誤（フロー） | `10_projects/<案件>/log/YYYY-MM-DD.md`（無ければ作成、あれば追記） |
| 確定した仕様・設計判断・"なぜそうしたか"の経緯（ストック） | `10_projects/<案件>/spec/` に新規ノート |
| 案件をまたぐ汎用知識・コマンド・トラブル対応 | `20_knowledge/` |
| 振り分け先が不明な走り書き | `00_inbox/` |

判断に迷ったら `00_inbox/` に置き、ファイル冒頭にどこへ振り分けるべきかのメモを残す。

## ノート作成時の必須フロントマター

```yaml
---
project: <案件名>        # 10_projects 配下のフォルダ名
type: log | spec | knowledge | inbox
status: active | done | archived
tags: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

テンプレートは `90_templates/` を参照。

## 書き方の原則

- **関連ノートは `[[ノート名]]` でリンクする**。バックリンク・グラフが知識ベースの価値の源泉。
- **ログは追記式**。既存の `log/YYYY-MM-DD.md` があれば上書きせず追記し、見出しで区切る。
- **案件 README（MOC）を更新する**。重要なノートを作ったら、その案件の `README.md` の目次に `[[リンク]]` を追加する。
- 相対日付は絶対日付に変換して書く（例: 「来週」→ 具体的な日付）。

## 触ってはいけない / 注意

- `90_templates/` のテンプレは勝手に書き換えない。
- 既存ノートを上書きする前に内容を確認する。説明と食い違う、または自分が作ったものでない場合は上書きせず指摘する。
